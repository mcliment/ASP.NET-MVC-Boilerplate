Comments are not allowed in the config.json file (See http://blog.getify.com/json-comments/ for why this is silly), so they are added here. The config.json file is responsible for holding application settings and options. It is the replacement for the appSettings configuration section contained in the web.config file in ASP.NET 4.6.
  • AppSettings - Stores general settings for the application e.g. the sites name.
    • SiteTitle - The name of the site.
    • SiteShortTitle - The short name of the site.
  • ApplicationInsights - Settings for application insights.
    • InstrumentationKey - The instrumentation key for Application Insights retrieved from Azure.
  • Logging - Settings for logging.
  • SitemapSettings - Stores sitemap related settings.
    • SitemapPingLocations - An array of URL's where the sitemap can be pinged to. Note that Yahoo uses the same sitemap ping location as Bing.
  • CacheProfileSettings - Stores the cache profile settings for all cached actions in the application. Note that the VaryByParam = "none" does not exist in MVC 6 yet and will be added when it becomes available.
    • BrowserConfigXml - Cache the browserconfig route (/browserconfig.xml) for a day.
    • Error - Cache the error route (/error/404/notfound) by status code for a day.
    • Feed - Cache the Atom 1.0 feed route (/feed) for a day.
    • ManifestJson - Cache the manifest route (/manifest.json) for a day.
    • OpenSearchXml - Cache the open search route (/opensearch.xml) for a day.
    • RobotsText - Cache the robots.txt route for a day.
    • SitemapNodes - Cache the nodes in the sitemap for a day.
